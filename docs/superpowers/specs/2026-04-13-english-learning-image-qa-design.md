# Design: English Learning Image Q&A POC

**Date:** 2026-04-13  
**Status:** Approved  

---

## Overview

A web application that helps users practice English by answering questions about an uploaded image. The system generates 5 mixed English questions (description + vocabulary + reasoning) using GPT-4o vision, conducts a conversational Q&A, then provides teacher-style feedback on the user's English.

---

## Architecture

### Stack

| Layer | Technology |
|-------|-----------|
| Frontend | React + TypeScript (Vite) |
| Backend | FastAPI (Python) |
| Agent Runtime | LangChain Deep Agents (LangGraph) |
| LLM | OpenAI GPT-4o (vision + text) |
| Session State | LangGraph MemorySaver (in-memory, POC) |

### Data Flow

```
React Frontend
  │  POST /sessions          (image upload)
  │  GET  /sessions/{id}     (current question)
  │  POST /sessions/{id}/answer (submit answer)
  ▼
FastAPI Backend
  ▼
Deep Agent (LangGraph Graph)
  │  analyze_image → generate_questions → ask_question ⟳ → evaluate
  ▼
OpenAI GPT-4o API
```

## Backend Design

### API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/sessions` | Create session, upload image. Returns `session_id` + first question |
| `GET` | `/sessions/{id}` | Get current session state (question, step) |
| `POST` | `/sessions/{id}/answer` | Submit answer. Returns next question or evaluation |

### Session Response Shape

```json
// Mid-session (steps 1-4)
{
  "session_id": "uuid",
  "step": 2,
  "total": 5,
  "question": "What do you think the people in the image are feeling?",
  "done": false
}

// Final response (step 5 answered)
{
  "session_id": "uuid",
  "step": 5,
  "total": 5,
  "evaluation": "Great effort! Your descriptions were clear and detailed...",
  "done": true
}
```

### Deep Agent Graph

The agent is a custom LangGraph `StateGraph` using the LangGraph runtime from Deep Agents:

```
START
  │
  ▼
[analyze_image]
  # GPT-4o vision: reads image, generates 5 mixed questions
  # Mix: ~2 description, ~1 vocabulary, ~2 reasoning questions
  │
  ▼
[ask_question]
  # Returns current question to API layer
  # Uses LangGraph interrupt() to pause and wait for user input
  │
  ▼
[receive_answer]
  # Stores answer in state, increments step
  │
  ├── step < 5 ──► [ask_question]   (loop back)
  │
  └── step == 5 ──►
                  [evaluate]
                  # GPT-4o evaluates all 5 answers together
                  # Tone: friendly teacher, no score/number
                  # Covers: grammar, vocabulary, fluency, content accuracy
                  │
                  ▼
                END
```

### State Schema

```python
class SessionState(TypedDict):
    image_b64: str        # Image encoded as base64 string
    questions: list[str]  # 5 questions generated by GPT-4o
    answers: list[str]    # User's answers (accumulated)
    current_step: int     # 0-indexed, 0–4
    evaluation: str       # Final feedback message (populated at end)
```

### Key LangGraph Pattern

`interrupt()` is called at the `ask_question` node to pause graph execution and hand control back to the API layer. When the user submits an answer via `POST /sessions/{id}/answer`, the graph resumes with the answer injected via `Command(resume=answer)`. This is the human-in-the-loop mechanism provided by Deep Agents / LangGraph.

## Frontend Design

### Screens

**1. Upload Screen**
- Image drag-and-drop or file picker (accepts JPEG, PNG, WebP)
- "Start Session" button — disabled until image selected
- On submit: calls `POST /sessions`, transitions to Chat Screen

**2. Chat Screen**
- Image thumbnail (top-left, fixed)
- Progress indicator: "Question X / 5"
- Message list: alternating agent questions and user answers
- Text input + submit button at bottom
- On submit: calls `POST /sessions/{id}/answer`
  - If `done: false` → append next question to message list
  - If `done: true` → transition to Evaluation Screen

**3. Evaluation Screen**
- Conversational feedback block from agent
- "Try another image" button → resets to Upload Screen

### Component Tree

```
App
├── UploadScreen
│   └── ImageDropzone
├── ChatScreen
│   ├── ImageThumbnail
│   ├── ProgressIndicator
│   ├── MessageList
│   │   └── MessageBubble (agent | user)
│   └── AnswerInput
└── EvaluationScreen
    └── FeedbackCard
```

### State Management

React `useState` + `useReducer` at App level — sufficient for POC scope. No Redux or external state library needed.

```typescript
type AppState =
  | { screen: 'upload' }
  | { screen: 'chat'; sessionId: string; step: number; messages: Message[] }
  | { screen: 'evaluation'; evaluation: string }
```

## Question Generation Strategy

GPT-4o receives the image and a system prompt instructing it to generate exactly 5 questions in a specific mix:

- **~2 Description questions** — "What do you see in the foreground?", "Describe the scene."
- **~1 Vocabulary question** — "What word would you use to describe the texture/color/mood of X?"
- **~2 Reasoning questions** — "What do you think is happening here?", "Why might the person be doing this?"

Questions are generated all at once during `analyze_image` and stored in state, then served one by one.

---

## Evaluation Strategy

After all 5 answers are collected, GPT-4o receives:
- The original image
- All 5 questions with their corresponding user answers
- A system prompt requesting conversational teacher-style feedback

The feedback covers:
- Grammar correctness (with specific corrections if needed)
- Vocabulary appropriateness
- Fluency and naturalness
- Accuracy of image descriptions

Tone: warm, encouraging, specific — like a language tutor, not an exam grader.

## Error Handling

| Scenario | Handling |
|----------|---------|
| Image too large (>5MB) | Frontend validation before upload |
| Invalid image format | Frontend validation + API 400 response |
| OpenAI API failure | API returns 502; frontend shows retry message |
| Session not found | API returns 404; frontend redirects to upload |
